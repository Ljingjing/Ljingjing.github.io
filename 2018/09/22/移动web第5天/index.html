笔记--移动web第五天

像素的概念

  1. 物理分辨率像素： 屏幕的分辨率（屏幕里面一层膜）
    1. 普通电脑 屏幕分辨率 1366*768
    2. 高级电脑 屏幕分辨率 1920 1080 2560 1660
    3. 普通手机 屏幕分辨率 750 * 1334
    4. 高级手机 屏幕分辨率 1125 * 2436
      屏幕分辨率值的是屏幕内屏里面的最小单元格
  2. 真实尺寸的像素 ： 真实尺寸的像素 固定的只要设备大小一样 真实像素就一样
    1. 普通电脑14寸 ： 真实 1366 * 768
    2. 高级电脑13.3寸 真实 1280 * 720
    3. 普通手机4.7 真实 375 * 667
    4. 高级手机5.5寸 真实 414 * 736w
      真实像素是屏幕外屏里面的最小的单元格

  3. 逻辑CSS的像素 在CSS里面设置px 网页的格子的最小单元

    1. 一般在移动端会设置视口 让网页的宽度=设备真实宽度
    2. 意味着网页的真实宽度和设备的真实宽度一致
    3. CSSpx 和 真实像素一致

  4. 在手机上由于物理分辨率和真实像素的大小不一样 一般2倍 更高有3倍的

    1. 写页面的时候参照是真实像素 页面在分辨率里面就放大2倍
    2. 如果是矢量的放大不会失真 如果图片非矢量图放大会失真
    3. 使用图片的时候要缩小一半来写 让他在分辨率放大2倍刚好和原图一样

移动端的JS

  1. 使用纯原生的JS 会使用HTML5新增的JSAPI
    document.querySelector(‘选择器’)
    document.querySelectorAll(‘选择器’)
    div.addEventListener(‘click’,function(){

    H5的APi使用方便 在移动端兼容性好

头部背景色渐变

  1. 当滚动条在滚动的时候 顶部搜索框的背景色需要渐变
  2. 在轮播图高度范围内进行渐变 超过轮播图范围后固定为1
  3. 在轮播图高度范围进行渐变 把背景色的透明度从0 到 1 的渐变过程
  4. 透明度值 就是使用滚动的距离/轮播图的高度

倒计时效果

  1. 要有一个时间倒计时
  2. 10点场 10点-12点 假如现在10.30 12.00-10.30 = 1.30
  3. 使用未来的时间 - 当前的时间 未来是12 - 当前 10.35 求到时间差的秒数
  4. 倒计时是每隔1秒总的时间–
  5. 把总时间时 分 秒 放到页面的span里面

轮播图效果

  1. 移动端需要自动轮播图
  2. 小圆点跟着自动轮播图
  3. 滑动轮播图
  4. 一般实现轮播图都使用插件实现

swiper轮播图插件的使用

  1. 下载swiper插件
  2. 下载完成后要解压 使用里面的dist目录
  3. 把dist复制到项目改成名字swiper
  4. 引包
    1. 引入swiper.css
    2. 引入swiper.js
  5. 写一个轮播图需要结构 
    <!-- 轮播图的图片容器 装的是所有的轮播图项(一般是图片) -->
<div class="swiper-wrapper">
    <!-- swiper-slide 每一个轮播项 -->
    <div class="swiper-slide">Slide 1</div>
    <div class="swiper-slide">Slide 2</div>
    <div class="swiper-slide">Slide 3</div>
</div>
<!-- 小圆点容器 -->
<div class="swiper-pagination"></div>
<!-- 左右箭头 -->
<div class="swiper-button-prev"></div>
<div class="swiper-button-next"></div>
<!-- 滚动条容器 -->
<div class="swiper-scrollbar"></div>

    注意一定要使用官网的类名因为有样式的没有类名就有样式

  6. 给轮播图大容器设置样式 限制轮播图的宽高
    .swiper-container {

    width: 600px;
height: 300px;

    }

  7. 添加一个JS代码去初始化轮播图 传入一些初始化的一些参数

swiper插件的一些常见参数

  1. 轮播图的方向
    direction: ‘vertical’ 垂直
    direction: ‘horizontal’ 水平
  2. 自动轮播图
    autoplay:{
    delay: 3000,//轮播图的延迟
    stopOnLastSlide: false,// 如果设置为true，当切换到最后一个slide时停止自动切换。（loop模式下无效）
    disableOnInteraction:true,//当用户滑动的时候禁止自动轮播图 不需要禁止就为false
    }
  3. 是否开启无缝轮播图
    loop: true,//是否开启无缝轮播图 开启无轮播图 如果不开 刷回去倒退回去
  4. 是否显示小圆点
    pagination:{
    el: '.swiper-pagination',//如果需要显示小圆点就指定小圆点容器
    }
  5. 是否要左右箭头
    navigation: {
    nextEl: '.swiper-button-next',
prevEl: '.swiper-button-prev',
    },//指定左右箭头的容器
  6. 是否要支持滚动条
    scrollbar: {
    el: '.swiper-scrollbar',
    },//指定滚动条的容器

写JS代码的规范

  1. 如果JS代码太多的时候不能全部写在一块 把每个代码区分功能块
  2. 当JS代码的功能不一样的时候要区分JS代码的功能
    1. 顶部是一个功能
    2. 倒计时是一个功能
    3. 轮播图也是一个功能
  3. JS使用函数来区分
    使用函数封装每个功能代码 方便调用 和管理每个功能 提高代码的阅读性和维护性
  1. 函数封装有2个作用
    1. 重复执行的代码 需要封装函数
    2. 区分功能块代码 也需要封装函数

  1. 再升华就是面向对象
    为什么要使用对象

    1. 对象只需要一个全局 减少全局污染
    2. 给函数归类 属于效果归在效果类 属于工具 归在工具类

  2. 对象再升华就是面向框架

    1. 可以把很多函数都需要的参数放到构造函数 原型对象里面的每个函数都能访问到公共的参数
    2. 把工具函数 功能函数放到对象的原型对象上

  3. 高级对象和框架式写法都是为了未来方便去维护和扩展代码

京东分类页面布局

  1. 京东分类页面是全屏的 水平和垂直都没有滚动条 （注意）

  2. 要显示更多内容 使用滑动来实现 （注意）
    左侧菜单有滑动
    右侧商品列表也有滑动
    滑动都是垂直方向

  3. 如果想要实现全屏页面 就要让body宽高和屏幕一样宽高 里面的内容也不能超出body 如果有超出要设置overflow:hidden

背景图的定位原点和背景图裁切

  1. background-origin: 改变背景图定位位置参照点 默认padding-box
    可以设置为border-box padding-box content-box
    改变后就会按照你设置的背景图定位参照盒模型范围的左上角

  2. background-clip: 实现背景图的裁切 默认 border-box 表示边框范围内都有背景图
    可以设置为border-box padding-box content-box
    设置了之后就会改变背景图出现的范围

  3. 通常2个会结合在一起使用 一般当盒子有padding或者有边框的时候才需要设置这些东西

实现分类页面的全屏布局

  1. 给头部 固定高度 位置固定定位在顶部
  2. 给底部 固定高度 位置固定定位在底部
  3. 给html和body设置高度100%
  4. 给body设置padding-top 和 padding-bottom 让main往中间挤
  5. 给main设置height:100% 就和body的高度一致
  6. 给main里面的category-left 和 category-right 设置高度100% overflow:hidden;

移动web

  1. 什么是移动web: 移动端(手机端)web(网页网站)的前端
  2. 为什么要学习移动web: 现代社会主流上网工具是手机 查看浏览网页基本使用手机 需要大量开发手机端的网页
  3. 怎么样学习移动web:
    1. 移动端网站m.jd.com m.taobao.com 移动端的网站和APP很相似 功能也很相似 有了web之后就有了移动App
    2. 移动和PC端对比：
      1. 移动web页面 内容比较少简单
      2. 移动web的页面大小针对手机的屏幕 图片文字比较小
      3. 移动web页面布局方式不一样 宽高不是写死需要去适配各种屏幕

移动的开发方式

  1. 响应式的开发方式： 一个页面兼容多个终端 一个页面同时在PC端和移动端都能够正常显示 屏幕宽度在变化的时候自动变化网页的布局
  2. 原生的开发方式： 针对每个端都写一个页面 PC端写一个PC端的页面 移动端写一个移动端的页面 如果你电脑访问打开PC端 如果是手机访问就打开移动端

响应式开发方式

  1. 响应式： 是一个页面同时兼容多个终端 随着屏幕宽度的变化而自动变化网页布局
  2. 响应式开发的原理： 使用百分比来写页面 同时根据屏幕的变化来改变元素宽度 早期使用JS来实现
  3. 使用响应式原理实现响应式布局变化
    1. 大屏幕的时候 一行放4个div 屏幕宽度>=1200 现代电脑
    2. 中屏幕的时候 一行放3个div 992 <= 屏幕宽度 < 1200 老款台式机
    3. 小屏幕的时候 一行放2个div 768 <= 屏幕宽度 < 992 ipad
    4. 超小屏幕的时候 一行放1个div 屏幕宽度<768 手机

响应式的开发框架

  1. 为什么要用框架： 框架封装好一系列的响应式功能 只要使用框架的类名就能实现响应式功能（框架把媒体查询已经封装好了）
  2. 常见的响应式开发框架
    1. bootstrap: 目前前端最流行的UI框架 应用最多最广泛 只针对UI
    2. MUI : 国内开发的响应式+移动端前端UI框架
    3. AmazeUI ：国内开发的响应式前端UI框架
    4. layUI ： 国内开发的响应式前端UI框架
  3. 使用框架的好处
    1. 方便快捷 快速开发响应式
    2. 框架的代码性能比较好 代码很规范
  4. 如果学好一个框架
    1. 阅读框架文档 知道常见有哪些组件 怎么用
    2. 使用框架的组件样式 在什么时候需要用
    3. 学会修改框架的内容

学习bootstrap框架

  1. 了解bootstrap历史： 2011年发布的框架 推特公司的前端开发一个UI框架
  2. 现在最新版本4.0 目前最稳定最流行的3.3.7版本

  3. 中文官网： https://v3.bootcss.com

  4. 学习步骤

    1. 下载bootstrap
      1. dist是生产环境 项目中用的框架代码
      2. 源代码作者开发bootstrap框架的时候他学的代码 源代码不是CSS是高级less或者sass 通过编译方式编译成普通CSS 源代码里面也会有dist使用源代码编译的代码
    2. 引包
      1. 引入css
      2. 引入JS
    3. 使用bootstrap的类名或者组件的代码

上午复习

  1. 移动web的概念：

    1. 针对手机端的网页
    2. 移动web站点m.jd.com m.taobao.com m叫二级域名
    3. 页面简单 内容比较少 图片文字相对比较小针对手机设备
    4. 手机端的兼容性少（所有浏览器内核webkit）

  2. 移动web的开发方式

    1. 响应式开发方式： 一个页面同时适配多个终端 通过屏幕宽度变化来改变布局
    2. 原生的开发方式： 针对每一个端都写一个页面 一般宽高都写死
    3. 响应式开发一般用在新建网站、对性能要求不是很高的网站
    4. 原生开发一般老网站 再次开发移动web就需要分开写 一些对性能要求特别高的也会使用原生开发

  3. 响应式开发原理

    1. 使用获取屏幕宽度来判断 设置样式的方式（代码太冗余 也很麻烦）
    2. CSS3媒体查询 相对比较方便 而且封装成类名 到处使用

    3. 媒体查询的语法
      @meida(条件) and (条件){

      条件成立执行的代码

      }
      width:值 width=值
      min-width:值 width>= 值
      max-width:值 width<= 值

      如果使用min-width作为条件 从小到大写 向下覆盖 （大屏幕吧小屏幕样式覆盖）
      如果使用max-width作为条件 从大到小写 向上覆盖 (小屏幕的样式把大屏幕样式覆盖)

  4. 响应式开发框架

    1. bootstrap
    2. mui
    3. amazeUI
    4. layui
      上述框架使用方式类似 只要学会一个bootstrap

  5. bootstrap框架的基本使用

    1. 下载包 下载生成环境的包dist目录的
    2. 引包
      引入bootstrap.css
      引入jquery.js
      引入bootstrap.js
    3. 去文档里面复制代码 粘贴 改一些内容即可使用

移动端的视口的概念

  1. 视口： 表示浏览器的可视窗口 相当于html的宽度范围
  2. 在PC端是不存在视口 视口没有问题的PC端的时候跟着浏览器宽度变化而自动变化
  3. 在移动端存在视口 存在问题： 移动端默认视口的宽度980px 主流移动端浏览器视口宽度都是980
    但是移动端手机屏幕宽度只有300-400 视口980需要在300多屏幕显示 需要缩放 内容包括图片等都会缩放
  4. 如何解决移动端端视口的问题： 把视口的宽度设置成和设备的宽度一样让视口不会缩放
    1. 宽度等于设备的宽度
    2. initial-scale=1 初始化缩放为1
    3. maximum-scale=1 最大缩放为1
    4. minimum-scale=1 最小缩放为1
    5. user-scalable=no 不允许用户缩放 手指拖放不了
    6. 完整视口 meta:vp 按tab键快速生成视口

栅格系统

  1. 分为row col-xx-列数 类似 表格tr 和 td
  2. 行必须放在container容器里面 和contaienr padding抵消
  3. 列必须放在row里面
  4. 列一行最多分为12列 通常使用栅格都会等比例 能被12整除的列
    1列 2列 3列 4列 6列 12列 注意没有小数
    公式就是 12总列数 / 要分的列数 = col-xx-数字
  5. 列分4个档次
    lg 大屏幕生效
    md 中屏幕生效
    sm 小屏幕生效
    xs 超小屏幕生效
  6. 特点
    1. 向下覆盖： 当同时写了多个类名 col-lg-3 col-md-4 会满足大屏幕吧小屏幕的类名覆盖
    2. 向上兼容 ： 只写一个或者2类名 只写了 col-md-3 但是没写lg 表示md和lg 列数一样的

原生的移动web

  1. 原生移动web: 单独给移动端写一套页面 页面内容简单 结构也简单 针对性强比较专一 运行效率性能高
  2. 常见原生移动web网站： m.jd.com m.taobao.com m.itheima.com 很多以m开头大部分都是原生的移动web
  3. 原生移动web的布局划分
    1. 头部
    2. 轮播图
    3. 导航菜单
    4. 广告 新闻
    5. 商品 或者一些信息列表
    6. 底部

  4. 原生移动web布局的方式

    1. 百分比布局 + 浮动
    2. 百分比 + flex
    3. rem + 浮动

    4. rem + flex （最流行最好的一种布局方式）

    5. 百分比： 给元素设置百分比的宽 只能实现宽度自适应

    6. rem : 是参照根元素(html元素)的字体大小 用当前的rem值*html的font-size 可以同时设置宽和高都使用rem 可以实现元素宽高都进行自适应
  5. 移动web都是在手机上使用： 手机内核都是webkit内核 兼容性问题很少 可以大量使用H5C3等技术来实现 移动web开发比较简单但是技术都比较先进 作为前端要不断学习

移动web开发的基本知识

  1. 像素

    1. 物理像素： 分辨率的像素
      iphone8屏幕：1334x750像素
      mac电脑分辨率：2560×1600
      国产电脑分辨率： 1366 768 19201080
      上述的参数都是物理像素(分辨率的px就是物理像素)
      手机1334是高 750是宽
      电脑2560是宽 1660是高
      电脑1336宽 768是高
      物理(屏幕分辨率)到底的什么：指的是屏幕里面的最小单元 是一根小格子 750水平能放750个格子 1334垂直能放下1334个格子
    2. 真实像素 ： 硬件英寸里面的真实的px
      设备硬件真实能够放的px数量
      真实像素和屏幕尺寸有关
      英寸是一个长度单位这个单位也是固定 1英寸 = 2.54
      英寸在手机里面表示手机对角线的长度
      真实像素： 就是在手机屏幕的英寸里面能够放下的像素数量
      14英寸 水平能够放下1366个px 垂直能放下768px
      4.7英寸 水平能放下375px 垂直能放下667px
    3. 逻辑像素 : CSS像素 在写页面的时候设置宽高px
      逻辑像素不是一个死的值根据视口来变化
      1. 例如在PC端访问 如果不加视口 PC端视口是和屏幕真实宽度一致 在PC端访问1CSSpx = 1真实px
      2. 例如在移动端访问 如果不加视口 1CSSpx != 1真实px 但是在移动端一般会添加视口让页面的宽度和屏幕的宽度一致 1CSSpx=1真实px
      3. 通常在开发的时候1CSSpx == 1真实px

  2. 物理像素和真实和逻辑像素的关系

    1. 物理像素分辨率 通常会比较大 电脑可以修改
    2. 真实像素和尺寸有关 是固定的

    3. 一般来说电脑分辨率和真实像素是一样的 但是有一些不一样
      普通电脑 1366分辨率=1366真实
      比例是1:1 1真实px = 1个物理px
      mac 但是有一些比较高 2560分辨率 = 1280真实
      比例2:1 1真实px = 4个物理px
      iphone678 750分辨率 = 375真实
      比例2:1 1真实px = 4个物理像素
      iphonex 1125分辨率 = 375真实
      比例是3:1 1真实px = 9个物理像素

      可以理解 就是比例越大的越好因为 用多个分辨率px格子来描述1个真实px

  3. 移动端二倍图的原理

    1. 因为移动端手机的分辨率通常 会 真实px要大2倍 目前主流手机分辨率都是真实2倍
    2. 由于分辨率是真实2倍 相当于 分辨率=CSSpx的2倍
      比如你写一个div宽度100px100px 在分辨率是使用200px 200px给你展示的
      字体大小是20px 在分辨率会变成40px
      图片是80px图片 变成160px
      页面中写CSS的所有内容 在分辨率里面都会放大2倍 一些字体图标 文字矢量放大了之后不会失真（移动端经常使用字体图标 因为放大不会失真）
      但是轮播图之类的也有图片图片放大之后就会失真
    3. 如果是2倍图移动端使用图片到时候 一般会设置图片的一半
      因为图片在分辨率会放大2倍 如果写375放大2倍 = 750 750刚好和原图一样
      但是如果你图片750 也写了750 放大2倍 变成1500 比原图大2倍所以失真
    4. 一般移动端图片的设计都是按照分辨率来设计的为了就是让图片在分辨率里面放大后不失真

总结移动端的概念

  1. 设备的物理分辨率像素和 真实 是不一样的
  2. 真实像素由于视口设置和页面宽度一样 真实像素和逻辑像素是一样的
  3. 通常手机端分辨率是比真实大2倍 电脑通常是1倍的
  4. 由于手机分辨率是真实2倍 页面在手机分辨率会放大2倍（内容就会变大 图片等也会变大 容易失真）
  5. 解决方案： 就是在手机端的时候设计图片按照分辨率来设计
    比如分辨率是750 图片宽度也是750
    写代码设置图片不能写750 要写750的一半375 等页面在分辨率放大2倍的时候375刚刚放大2倍=750
    以后只要看到移动端的图片都缩小一半来写图片标签 还是背景图

jd头部布局

  1. 左边一个logo图标
  2. 中间一个from表单 里面一个search图标 和 Input输入框

  3. 右边一个登录链接

  4. 样式 给图标设置背景图 同时设置背景大小 使用flex左右2边固定 中间flex:1自适应

轮播图布局

  1. 暂时只添加一个图片 后面在做轮播图

  2. 样式 给图片设置宽度百分百

导航条布局

  1. 写一个列表 里面8个li 里面都有a 和 图片旁边有一个p段落

  2. 样式 给ul设置flex布局同时支持换行 每个li宽度25% 里面的a转块设置居中 图片设置原图的一半

主体商品布局

  1. 写一个main大容器包裹所有商品
  2. 在main里面添加一个秒杀商品容器

  3. 再添加3个京东超市容器

  4. 给main设置一个背景色 和 padding:10px;

秒杀商品布局

  1. 添加秒杀的标题title
  2. 添加秒杀的商品内容content
  3. 在title里面添加秒杀的图标 文字 倒计时 更多

  4. 在content 添加 ul li 里面添加a 图片 和价格

  5. 样式 给title设置flex布局 给图标 文字 倒计时设置间距 让更多定位到右边
    给content的ul设置flex布局 每个li flex:1 设置里面图片宽度缩小一半

京东超市布局

  1. 添加一个京东超市title
  2. 添加一个京东超市内容content
  3. 在title里面添加一个h4标题
  4. 在content内容里面添加 左left 右 right 容器

  5. 在左右容器里面 添加a a里面添加图片

  6. 样式 给京东超市title设置行高 左padding 再添加一个伪元素定位到左边
    给内容设置flex布局 左右容器flex:1;
    给a转块 图片100%

底部的布局

  1. 添加一个ul li

  2. 在li里面添加a a 里面添加图片 图片地址使用(检测元素查看完整版网络地址)

  3. 样式 给ul设置flex布局 li 设置flex:1 里面a转块 文本居中 图片缩小一半设置宽高

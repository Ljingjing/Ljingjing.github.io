元素宽高继承问题

  1. 父元素宽度会继承给块级子元素
  2. 但是父元素的高度是不会继承给子元素的
  3. html 和 body 和 window的关系和他们的高度
    window 浏览器窗口
    html整个网页的文档
    body整个网页内容的文档
    由于手机屏幕浏览器的高度和网页高度一致
    window在手机端和html和 body重叠的
    window最大 高度和屏幕高度一样
    html是window的儿子 由于高度不会继承 你可以设置高度100% 可以和window一样高
    body是html的儿子 虽然html有高了但是不会继承给body body没有高 如果要高也要设置h100%和html一样高

jd分类左右滑动的功能

  1. 有一个父容器就是category-left
  2. 有一个子容器 ul
  3. 父容器高度固定 整个页面高度-头部-底部高度
  4. ul高度不是固定 由li内容撑开
  5. ul高度比父元素高 为了看到后面的内容 实现滑动来看

实现左侧分类的点击效果

  1. 点击了某个导航链接要 位移到让当前点击的链接吸顶
  2. 相当于就是点击了某个链接位移 到当前点击的li的地方 位移的距离就是-li的索引li的高度
    第一个li 索引0 - 0     46 = 0px;
    第二个li 索引1 - 1 * 46 = -46px;
  3. 但是到了后面的链接已经到达最大位移距离就设置为最大位移的距离
    当父元素和子元素底部相连的时候就是最大位移的距离
    最大距离 = 父元素的高度 - 子元素的高度 600px - 1700px = -1100px

移动端的屏幕适配

  1. 移动端手机屏幕大小不一
    320
    360
    375
    414

  2. 为了让我们的网页兼容各种屏幕 在移动端需要做屏幕适配 手机网页都是全屏的不像PC端居中显示

  3. 怎么实现屏幕适配

    1. 百分比布局 根据屏幕宽度自动变化百分比 宽度百分比 高度写死（高度无法百分比）
    2. flex布局 也可以实现宽度自适应 但是也不能实现高度自适应
    3. rem布局 既可以实现宽度自适应 又可以实现高度自适应 （整个网页全部自适应）

使用Rem实现屏幕适配

  1. Rem是什么： root element 根元素元素 是一个相对单位 参照根元素的字体大小 根元素就是html元素只有祖先
    1rem = 1根元素的字体大小 也是一种单位跟px一样的可以在任意写px地方用rem
    默认网页字体是16px html字体大小就是16px
    默认1rem = 16px 10rem = 160px
    如果你改了html根元素字体大小 使用的rem也会跟着变化
    但是注意网页最小是字体大小是12px 根元素字体大小最小12px

  2. em: element 元素 是一个相对单位 相对的是当前元素的字体大小 参照的是自己
    默认当前元素字体大小是16px 1em=16px
    如果修改了当前元素的字体大小那么em也会修改

  3. 相同的rem和em都是相对单位 相对的都是字体的大小

  4. 实现屏幕适配一般使用rem不使用em

    1. rem 参照根元素 所有rem都是统一的大小
      只要修改了根元素的字体大小整个页面用到rem都会变化 只需要该一个值实现屏幕适配
    2. 使用em 参照自身 自身字体大小不均与 导致em大小也不样 如果修改 可能每个都要
  1. 使用rem实现屏幕适配的原理
    1. 指定一个标准屏幕 比如iphone 678 375 指定标准屏幕根元素字体大小比如100px
    2. 页面所有使用px地方换成rem 刚好缩小100倍来写 width:100px = width:1rem
    3. 还需要实现所有屏幕都会参照标准的375来走
      320 根元素字体大小小一点
      414 根元素的字体大一点
    4. 变化的计算公式
      当前屏幕宽度 / 标准屏幕的整屏的rem值
      320 / 3.75 == 85.333
      414 / 3.75 == 110.4
      使用JS来获取屏幕宽度 / 标准3.75
      把这个计算好的当前屏幕根元素字体大小设置给当前根元素
      如果需要屏幕变化拉动自适应 添加resize事件
  1. rem 工具的使用 http://alurk.com/
    1. 帮你把px转成rem 只需要把代码复制进去 转好粘贴回来即可
    2. 把标准根元素字体大小设置为100x
    3. 把body的字体大小设置为16px要还原
    4. 引入当前使用JS 动态变化根元素字体大小的js 要在head中引入影响到页面的渲染

找bug技巧

  1. 多看bug(见识广)

  2. 会找bug

    1. 页面结构bug 标签是否有结束 标签是否会多加结束 标签单词错误
    2. 页面样式bug
      1. 样式没出来
        1. 样式文件没引入
        2. 样式生效没有（审查元素找到要设置样式的元素） 如果看不到 选择器选错 类名或者id写错
      2. 样式有但没有效果
        1. 样式属性是否写 属性前有感叹号属性错了 或者值
        2. 样式属性是否被覆盖或者继承 样式属性有没有中划线
      3. 自己不知道怎么写样式 居中（内容居中text-align:center）块居中margin:0 auto;
      4. 引入bootstrap会出的样式问题
        1. 没有引入包
        2. 类名错误
        3. 发现覆盖不了bootstrap选择器优先级不够
    3. 页面功能bug
      1. 功能没生效
      2. 文件没引入
      3. 依赖的文件没引入（jquery zepto ）
      4. 是否定义函数 有没有被调用
      5. 代码是否报错 （单词错 赋值错 变量名。。）
      6. 看代码是否执行 （断点调试看看代码是否执行（事件没被触发 事件名错误 元素没获取到））
      7. 一些获取值的方式错误（兼容性问题 单词错误）
      8. 逻辑错误 (代码执行顺序是否符合你写的顺序 )
      9. 变量名重复 全局已经有变量 局部又用var
    4. 环境系统bug

  3. 善于总结

    1. 出了bug记录下来 bug现象 bug原因 解决方案

移动web总结

  1. 移动web开发的现状： 前端最热门的开发 市场大 需求大 工资高 代码少 兼容性少 容易学
  2. 移动web的开发方式
    1. 响应式开发方式 一个页面适配多个终端
    2. 原生开发方式 单独PC和移动端都写一套代码

  3. 响应式开发的原理： 媒体查询
    @media(width:值){
    //条件成立执行的代码
    }
    判断有3种
    width:值 等于这个值
    min-width:值 大于等于这个值 从小到大写 向下覆盖
    max-width:值 小于等于这个 从大到小写 向上覆盖

  4. 响应式的开发框架
    bootstrap 最常用的
    MUI
    AmazeUI

  5. bootstrap的使用

    1. 下载包
    2. 看懂文档 全局CSS样式（一些简单类名） 组件 （标签类名组合在一起的效果但是没有交互） 插件 （组件并带有交互）
    3. 找到需要的样式或者组件插件 复制结构
    4. 需要修改样式 审查元素找到样式对应的类名 覆盖 推荐外面使用id 方便覆盖

  6. 容器 container 布局容器 居中 栅格系统 行和 列 row放到container里面 col放到行里面
    列有4个档次 xs sm md lg
    一行最多分为12列 col-xs-1 col-xs-12

  7. 导航条 轮播图 折叠菜单collapse 标签页 表单表格 媒体对象 弹出框 提示工具 模态框 按钮 响应式工具

  8. 字体图标上传svg图标下载代码引入css使用类名 原理是使用CSS3 web字体 引入对应字体文件 使用图标对应的编码

  9. 移动端基本概念 手机分辨率通常比真实宽高大2倍 图片都是按照分辨率来设计 页面CSS宽高是按照真实的宽高设置 使用图片和背景图要缩小一半来写

  18. 移动端点击事件 有300ms延迟 可以使用touch事件模拟click 也可以使用第三方包fastclick解决

  19. 常见移动端滑动事件 touchstart touchmove touchend e事件源对象 e.touches所有手指
    e.touches[0].clientX 手指在页面上水平位置 e.touches[0].clientY 垂直位置
  20. 移动端zepto库 类似jquery 有jquery就不要使用zepto 不要同时引入2个包 $ 会冲突
  21. rem: root element根元素 参照根元素html元素的字体大小 1html的字体大小16px 1rem=16px
  22. rem的好处 相对html字体大小html字体大小不会随意变化 统一所有使用rem的大小 实现宽度和高度的和内容都自适应
  23. 使用rem实现网页自适应的原理是通过媒体查询 或者JS来不断改变html的字体大小html字体大小改变 那么使用rem单位大小都会被改变
  24. rem工具 http://alurk.com/ 使用 把px转成rem按照750设计稿调试的时候也是按照750屏幕调试
    把页面做的跟设计稿一模一样 把px转成rem并且html字体大小也跟着变化 会在所有屏幕都进行等比例缩放 整个页面自适应 注意改了html字体大小 整个页面都会继承html字体 文字可能会很大 把body字体大小设置回16px
